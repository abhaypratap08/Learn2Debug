Learn2Debug upgrade context
Date: 2026-04-12

Purpose
This file captures the current upgrade work and the relevant repo state so the next session has context without re-scanning everything.

Main upgrade implemented
1. The backend analyzer is now level-aware instead of treating `level` as summary-only metadata.
2. Supported analysis levels are `beginner`, `intermediate`, and `advanced`.
3. The API response now returns the effective `level` and `checksApplied`.
4. The frontend now lets the user choose the analysis level and displays the returned level/check count.

Backend files updated
- src/main/java/com/learn2debug/model/AnalysisResponse.java
  Added `level` and `checksApplied` to the response record.

- src/main/java/com/learn2debug/service/CodeAnalysisService.java
  Added real level gating.
  Added or upgraded checks for:
  * division by zero, including tracked zero-valued variables
  * null dereference from tracked null state
  * String reference comparisons using `==` and `!=`
  * assignment in `if`/`while`/`do-while`/`for` conditions
  * empty control-flow bodies caused by stray semicolons
  * constant boolean conditions
  * raw collection types
  * unused local variables
  * overly broad catch clauses
  * empty catch blocks
  * mutable public fields
  Parsing was also upgraded so bare snippets are wrapped consistently.
  Important parser detail: JavaParser can parse snippet-like input as a Java 21 compact class named `$COMPACT_CLASS`; the service now detects that case and still wraps the code so flow-sensitive checks run correctly.

- src/test/java/com/learn2debug/controller/AnalysisControllerTest.java
  Test coverage was extended for:
  * returned `level` and `checksApplied`
  * String `!=` reference comparisons
  * tracked zero denominator detection
  * intermediate-only checks
  * advanced-only checks
  * invalid level fallback to `beginner`

Frontend file updated
- frontend/app/page.tsx
  Added analysis level types and selector UI.
  Request body now sends the selected `level`.
  Results UI now shows returned level and applied check count.

Verification completed
- `mvn test` passed
- `npm run lint` passed
- `npx next build --webpack` passed

Known environment-specific issue
- `npm run build` using Turbopack fails inside the current sandbox because Turbopack tries to create/bind a local port during CSS processing and receives `Operation not permitted`.
- This was treated as an environment limitation, not as an app regression.
